# Cache key design

Cache keys are generated by `CacheKeyGenerator` in `src/services/cache.ts`.

| Data | Format |
|---|---|
| Search | `search_<keyword>_<page>_<pageSize>` |
| Nodes | `node_<category>_<parentId-or-root>` |
| Metrics | `metric_<setId>_<name-or-all>` |
| Series | `series|<setId>|<metricIds>|<periods>|<areas>` |
| External source | `datasource_<source>_<sorted-params>` |

Series keys use `|` between segments and `,` inside collection segments. Every value is encoded with `encodeURIComponent` after collection values are sorted. If no area is supplied, the final segment is `000000000000`.

Example:

```text
series|A01|metric-a,metric-b|2023YY,2024YY|000000000000
```

The previous underscore-only format could not distinguish `metricIds=[a,b], periods=[c]` from `metricIds=[a], periods=[b,c]`. The segmented format fixes that collision. Existing old-format entries naturally expire and are not migrated.
